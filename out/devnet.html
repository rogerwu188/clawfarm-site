Implemented capabilities

Receipt-driven settlement

The active Phase 1 flow records one economic settlement per attested receipt instead of using the removed epoch settlement model.

Provider USDC stake

Providers register one wallet and post the configured USDC stake into a dedicated provider stake vault.

Treasury and pending-revenue vaults

Receipt recording splits user-paid USDC into treasury USDC and provider pending-revenue USDC vaults.

Finalized provider payout

Provider USDC is released from the pending vault only after attestation finalizes the receipt.

CLAW reward accounting

User and provider CLAW rewards are snapshotted per receipt, tracked as pending, locked, released, and claimed balances.

Challenge economics

Challenges use CLAW bonds and can return the bond, refund the payer provider-share USDC, slash provider CLAW position, reward challengers, and burn CLAW.

AIRouter model catalog

AIRouter can aggregate provider model catalogs from models_url, cache normalized model metadata, and expose provider-prefixed model IDs.

AIRouter routing profiles

The runtime supports service-side clawfarm/eco, clawfarm/auto, and clawfarm/premium routing with capability filtering, scoring, and fallback attempts.

Current limitations
01No user pre-funded PDA escrow flow is implemented in the current masterpool instruction surface.
02No user self-service balance exit flow is implemented for a pre-funded escrow balance, because that balance model is not part of Phase 1.
03No published ClawFarm JavaScript SDK package is confirmed from this repository set or the npm registry.
04No self-serve on-chain model price table exists in the current provider account; model pricing lives in AIRouter provider catalog/cache configuration.
05No automated daily reward release executor is included; Phase 1 has reward accounting and claim/release primitives.
06The current Phase 1 default USDC split is provider 30% and treasury 70%, not the mainnet-target provider-majority split.
07SolanaService production enablement cannot be confirmed from repository defaults; the default AIRouter config sets it disabled.
